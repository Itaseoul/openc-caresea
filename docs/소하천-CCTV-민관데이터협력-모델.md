# 소하천 환경감시 CCTV 민관 데이터 협력 모델 (검토 메모)

작성 2026.06.29. 외부 분석(의뢰인 제공)을 우리 사업 맥락에서 검증하고 교정한 문서. 문체는 docs 규칙을 따른다(서술형, 하이픈 미사용, 보수적 수치, 정직 경계).

핵심 발상은 이타서울이 소하천에 환경감시 CCTV를 설치하되 단순한 투기 감시 카메라가 아니라 하천 유입 폐기물 인공지능 실증과 공공데이터 전환 사업으로 접근하는 것이다. 이타서울이 미션클리어와 이타시티에서 쌓은 쓰레기 종류와 수거 위치와 지리공간 데이터 정제 경험이 이 접근의 근거가 된다. 이 방향은 우리가 앞서 정리한 공공 투자의 공백, 곧 영상 기반 안전 정보와 부유물 포집을 메우는 데이터 생산자 역할과 정확히 맞물린다. 관련 분석은 docs 공공자료 폴더의 공공기관 관심사 분석을 참조한다.

## 1. 두 가지 사업 방식

| 구분 | 방식 A 공유재산 기부채납 | 방식 B 민관 공동 데이터 협약 |
|---|---|---|
| 개념 | 이타서울이 장비를 사서 설치한 뒤 소유권을 지자체에 무상 기부 | 이타서울이 하드웨어를 소유하고 운영하며 지자체와 MOU로 데이터 공동 활용 |
| 운영자 | 지자체가 영상정보처리기기 운영자로 등록 | 이타서울이 운영 주체 |
| 데이터 | 이타서울이 API나 정기 리포트로 공유받음 | 이타서울이 비전 인공지능으로 가공해 주도적으로 축적하고 이타시티와 정부에 제공 |
| 부담 | 유지보수와 전기료와 회선비와 개인정보 보호 책임을 지자체가 부담 | 운영과 책임을 이타서울이 부담하되 데이터 주도권 확보 |

## 2. 검증된 법정 절차

소하천 구역에 고정형 CCTV 지주를 설치하려면 관할 지자체의 소하천 담당 부서에서 소하천 점용허가를 받아야 한다. 근거는 소하천정비법 제14조이며 관리청은 시장과 군수와 구청장이다. 처리기간은 신청 접수 후 20일이고 그 기간 안에 통지가 없으면 허가가 의제된다. 국가하천이나 지방하천으로 흘러드는 소하천은 관리청이 해당 하천관리청에 통지하고 환경부에 보고한다. 이 부분은 외부 분석의 법 인용이 정확함을 국가법령정보센터 기준으로 확인했다.

공개된 장소에 CCTV를 설치하기 전에는 개인정보 보호법 제25조에 따라 주민 의견 수렴이 필요하다. 지자체와 협약한 뒤 지자체 홈페이지에 20일 이상 행정예고를 게시하는 방식이 깔끔하다.

프라이버시 보호 설계는 이타서울의 기존 기조와 일치한다. 쓰레기 수거 상황과 하천 수위 같은 환경 데이터만 추출하고 영상에 찍히는 얼굴과 차량 번호판은 엣지 단이나 서버에서 즉시 마스킹한다. 안내판에는 운영 주체와 공동 활용 기관을 함께 명시한다.

## 3. 지자체 설득 역제안

부서 간 칸막이로 민간 제안이 미뤄질 수 있으므로 민원이 아니라 데이터 가치로 접근한다.

| 지자체 우려 | 역제안 |
|---|---|
| 관제 인력이 부족하다 | 실시간 관제를 요청하지 않는다. 투기 발생 시각과 투기량과 빈도만 시각화 데이터로 정제해 환경과와 하천과에 제공한다 |
| 개인정보 유출 책임이 우려된다 | 원본을 그대로 저장하지 않고 얼굴과 번호판을 마스킹한 환경 정보 위주 데이터만 추출한다 |
| 설치와 유지 예산이 없다 | 설치비와 초기 운영비를 민간 기금으로 부담하므로 지자체 예산은 들지 않는다 |

## 4. 비판적 각주와 교정

외부 분석은 방향이 옳지만 그대로 받기 전에 다음을 바로잡는다.

첫째, 법적 리스크가 없다는 표현은 과장이다. 방식 B로 우리가 직접 운영하면 우리가 영상정보처리기기 운영자가 되어 개인정보 보호법상 의무의 주체가 된다. 마스킹은 위험을 크게 낮추지만 없애지는 못한다. 운영자 책임이 지자체로 넘어가는 것은 방식 A 기부채납에 한정되며 그조차 위험이 사라진다기보다 이전된다고 보아야 한다.

둘째, 방식 선택은 우리 가치와 맞물린 결정이다. 방식 B는 데이터 주도권과 인공지능 실증에 유리하지만 청년 단체에 운영과 법적 부담이 무겁다. 방식 A는 부담이 가볍지만 하드웨어 소유와 데이터 흐름을 지자체에 의존한다. 권고는 하이브리드다. 실증 기간에는 MOU로 우리가 운영하며 인공지능을 실증하고, 지속 가능성 출구로 기부채납을 설계해 장기 책임을 지자체로 옮긴다.

셋째, 자체 CCTV 설치는 신규 스코프다. 지금까지 우리 접근은 국가교통정보센터와 홍수통제소 같은 공개 피드를 모으는 것이었다. 자체 CCTV는 하드웨어와 지주 점용허가와 행정예고와 프라이버시 설계를 새로 요구한다. 두 갈래를 단계로 분리한다. 지금은 공개 피드 집계로 즉시 저비용으로 운영을 보이고, 파일럿 한 곳에 자체 인공지능 CCTV를 두어 차별화한 데이터 생산자로 선다.

넷째, 인공지능 쓰레기 검출은 실증이지 보장이 아니다. 자동 정확 집계를 약속하지 않고 존재와 추세를 보여주는 보조 수단으로 둔다.

다섯째, 법정 등급의 불일치를 정리해야 한다. 소하천 CCTV라는 이름과 달리 우리 주 후보지 괴정천은 지방하천이고 사하구의 지정 소하천은 없다. 지방하천 점용허가는 소하천정비법 제14조가 아니라 하천법 제33조를 적용한다. 따라서 실제 대상 하천이 법정 소하천인지 지방하천인지에 따라 인용 조문과 처분 절차가 달라진다. 입지의 법정 등급 확인이 선행 과제다.

## 5. 우리 케이스 권고 요약

실증 기간에는 MOU 방식으로 운영하며 인공지능과 데이터 공공 환원을 실증하고 지속 단계에서 기부채납으로 책임을 이전한다. 공개 피드 집계는 지금 바로 운영하고 자체 인공지능 CCTV는 파일럿 한 곳으로 좁힌다. 인용 조문은 대상 하천의 법정 등급으로 분기한다. 지방하천이면 하천법 제33조이고 소하천이면 소하천정비법 제14조다. 데이터와 프라이버시 설계는 이타서울의 기존 무수집 기조와 마스킹 우선 원칙을 그대로 적용한다.

## 6. 우리 자산과의 연결

permit 에이전트는 현재 하천법 제33조 하천 점용허가만 생성한다. 이 모델을 반영하면 법정 등급 분기, 곧 소하천정비법 제14조와 하천법 제33조를 가르고, CCTV 점용허가와 개인정보 보호법 제25조 행정예고와 프라이버시 안내판과 민관 협약 또는 기부채납 문서팩을 추가하는 확장이 자연스럽다.

ChangeX 응모서는 데이터 생산자와 민관 데이터 거버넌스와 비전 인공지능과 프라이버시 바이 디자인을 거버넌스와 측정 항목에 보강할 수 있다.

## 7. 다음 확인

실제 대상 하천의 법정 등급을 고시로 확정한다. 관할 지자체의 하천관리과와 환경과와 정보통신과 담당 부서를 특정한다. 인공지능 쓰레기 검출의 실증 범위와 한계를 정직하게 정의한다.

## 8. 대상지 발굴 프레임 (오염 유형별)

어디에 설치할지를 정하는 틀이다. 의뢰인 제공 분석을 검증해 정리했다. 정책 배경으로 해양수산부가 관계 부처와 함께 2027년까지 하천을 통한 바다 쓰레기 유입을 2023년 대비 삼할 이상, 곧 6.5만 톤 수준으로 줄이는 저감 대책을 세웠다는 사실을 korea.kr 정책브리핑으로 확인했다. 바다와 직접 이어지는 권역의 작은 하천이 지자체의 해결 욕구가 가장 높은 곳이다.

유형은 넷이다. 첫째 해양 유입 차단 권역으로 한강 권역의 김포와 강화 일대, 영산강 권역의 담양 일대, 낙동강 권역의 부산 산업단지와 주거지를 지나는 하천이다. 둘째 농어촌의 불법 투기 사각지대로 경기와 충청 외곽의 화성과 평택과 아산 일대다. 셋째 휴가철 피서지 하천으로 강원과 경기의 계곡 주변과 제주 도서 지역이다. 넷째 구도심 주거지의 복개 지류로 인천 미추홀구와 서구, 서울 일부 지류다.

발굴 방법은 세 단계다. 환경부 물환경정보시스템의 수질 지표와 국가수자원관리종합시스템의 유역 통계를 연계해 수질이 급격히 나빠지는 구간을 일차로 거른다. 공공데이터포털에서 지자체별 무단투기 단속 CCTV 현황과 폐기물 방치 정화 실적을 받아 인프라가 가장 부족한 사각지대를 찾는다. 발굴한 핫스팟의 하천관리과에 비식별 인공지능 모니터링 모델을 역제안하면 지자체의 해양폐기물 저감 국비 사업 실적과 연계되어 승인이 빨라진다.

다만 두 가지를 분명히 한다. 첫째 ChangeX 응모는 부산으로 한정되므로 일차 진입점은 낙동강 권역의 부산 하천, 곧 감전천과 장림천이고 나머지 유형은 검증 이후 복제 지도로 둔다. 다만 감전천은 지방하천이고 장림천은 복개가 많은 기타하천이어서 대상 하천의 법정 등급과 복개 여부를 먼저 확인해야 한다. 둘째 투기자를 적발하는 야간 감시 프레임은 개인정보와 법적 위험을 키우므로, 환경 데이터를 정제해 제공한다는 프레임을 앞세우고 비식별 설계를 유지한다.
