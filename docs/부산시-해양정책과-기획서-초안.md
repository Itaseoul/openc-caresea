# 부산시 해양정책과 제출용 기획서와 데이터 흐름도 초안

작성 2026.06.29. 이 문서는 부산시 해양정책과 제안 메일에 붙일 기획서와 시스템 아키텍처 초안이다. 외부 제안 자료를 검토해 과장과 미검증 표현을 교정했고, 교정 내역은 끝의 검토 메모에 정리했다. 메일 본문은 `docs/부산시-해양정책과-제안-메일초안.md`를 함께 본다.

문체는 공공 행정 문서에 맞추되 과장하지 않고, 확정되지 않은 항목은 확인 필요로 표시한다.

## 1. 시스템 데이터 흐름도 (4단계 아키텍처)

원본 영상이 개인정보를 남기지 않고, 환경 데이터만 정제되어 두 플랫폼으로 전달되는 구조다.

```
[1단계 현장 수집]        [2단계 엣지 AI 비식별]            [3단계 클라우드 허브]      [4단계 최종 활용]

 ┌────────────┐        ┌──────────────────────┐         ┌──────────────┐        ┌────────────────────┐
 │ 소하천 합류부 │        │  현장 제어함 엣지 장치   │         │              │ ─ API ▶│ 부산시 해양환경     │
 │ 고정형 CCTV  │ ─RTSP▶ │ - 얼굴·번호판 실시간    │ ─VPN▶  │  이타서울     │        │ 데이터 플랫폼       │
 │ (IP 카메라)  │        │   마스킹(서버 저장 전)   │ 암호화  │  클라우드     │        │ (연계 협의 대상)    │
 └────────────┘        │ - 부유 폐기물 객체 검출  │  전송   │ (비식별 데이터)│        └────────────────────┘
                       │   (플라스틱·스티로폼 등) │         │              │ ─ 웹 ─▶│ 이타시티(ita.city) │
                       └──────────────────────┘         └──────────────┘        │ 시민 공개 시각화    │
                                                                                └────────────────────┘
```

단계별 처리 표준은 다음과 같다.

1단계 수집. 낙동강으로 이어지는 소하천 합류부 길목에 고정형 IP 카메라를 설치하고 영상 스트림을 발생시킨다.

2단계 비식별과 검출. 현장 제어함의 엣지 장치에서 영상을 실시간으로 처리한다. 사람과 차량이 감지되면 서버에 저장하기 전에 얼굴과 번호판을 마스킹하여 개인 식별 요소를 현장에서 제거한다. 동시에 부유 폐기물의 존재와 이동을 검출한다. 검출은 실증 단계이며 자동 정확 집계를 약속하지 않고 존재와 추세를 보여주는 것을 목표로 한다.

3단계 적재. 암호화된 보안망으로 개인정보가 제거된 환경 분석 데이터와 메타데이터만 이타서울 클라우드로 전송한다. 영상 원본은 서버에 저장하지 않거나 최소한으로만 보관한다.

4단계 활용. 부산시에는 소하천 폐기물 유입의 시각과 추정량과 성상 추세 통계를 공개 자료나 API 형태로 제공한다. 이타시티에는 하천 오염 추세를 지도로 시각화하여 시민과 연구자가 보고 정화 활동으로 잇도록 한다.

## 2. 기획 제안서 초안

### 1. 사업 개요

사업명은 인공지능 비식별 모니터링 기반 소하천 유입 폐기물 추적과 데이터화 실증 사업이다. 제안자는 사단법인 이타서울이다(대표자 성명은 발송 전 확인). 제안 대상은 부산광역시 해양농수산국 해양정책과다. 사업 기간은 협약 체결일로부터 1년이며 협의로 연장할 수 있다. 사업 대상은 낙동강 하구로 이어지는 핵심 소하천 합류부 〔개소수〕 곳으로 사하구와 사상구와 강서구 가운데 협의로 정한다. 규모는 약 열 대 내외의 작은 실증으로 둔다.

### 2. 추진 배경과 필요성

부산 연안과 다대포 일대로 들어오는 해양쓰레기의 상당 부분이 낙동강을 비롯한 육상 하천에서 기인한다(정확한 비율은 부산시와 해양수산부 자료로 확정). 바다로 들어온 뒤 수거하는 방식은 비용이 크고 효율이 낮으므로, 유입 직전 단계인 소하천 길목에서 모니터링하고 선제적으로 대응하는 인프라가 필요하다. 해양수산부와 관계부처는 2027년까지 하천을 통한 바다 쓰레기 유입을 2023년 대비 삼할 이상 줄이는 저감 대책을 세웠고, 부산시는 드론과 인공지능 기반 해양쓰레기 모니터링을 선도하고 있어, 육상 기인 유입량 데이터의 실시간 공급원이 이 흐름에 맞닿는다. 2028년 제4차 UN 해양회의가 우리나라에서 열리고 부산이 개최도시 유치를 추진하는 점도 시의성을 더한다.

### 3. 핵심 사업 내용과 기술적 차별성

엣지 단 실시간 비식별 처리를 적용한다. 개인정보보호위원회 고정형 영상정보처리기기 가이드라인을 준수하도록 설계하여, 현장 엣지 단에서 얼굴과 차량 번호판을 실시간으로 마스킹한 뒤 환경 데이터만 추출한다. 영상 원본을 서버에 저장하지 않아 개인정보 유출과 민원의 소지를 크게 낮춘다. 다만 시스템을 운영하는 주체는 개인정보 보호법상 영상정보처리기기 운영자의 의무를 지므로 운영 관리 방침 수립과 안내판 게시를 함께 이행한다.

부유 폐기물의 종류와 시간대별 이동을 인공지능 객체 검출로 데이터화한다. 이는 실증이며 정확한 자동 집계를 약속하지 않고 존재와 추세를 보여주는 보조 수단으로 둔다.

공공과 민간의 데이터 양방향 허브를 구축한다. 부산시에는 소하천 폐기물 유입량과 이상 징후 데이터를 제공하고, 시민에게는 이타시티 플랫폼으로 하천 환경 데이터를 시각화하여 시민 주도 정화 활동을 유도한다. 부산시 해양환경 데이터 플랫폼과의 연계는 명칭과 연계 사양을 확인한 뒤 협의로 진행한다.

### 4. 기관별 역할 (민관 협력)

부산광역시 해양정책과와 관할 구청은 CCTV 설치를 위한 소하천 점용 허가와 행정 협조, 개인정보 보호법에 따른 주민 의견수렴 행정예고의 지자체 홈페이지 게시, 부산시 해양환경 시스템과의 데이터 연계 협조를 맡는다.

사단법인 이타서울은 카메라와 엣지 장치 등 하드웨어 인프라를 전액 투자하고 설치하며, 인공지능 구동과 데이터 정제와 시스템 유지보수와 클라우드 운영을 전담하고, 정제된 모니터링 데이터 분석 리포트를 정기 제공한다. 설치와 운영비는 민간이 부담하며 일차 자본은 마이크로소프트가 지원하는 환경 기금에 신청 중이다.

### 5. 기대 효과

유입 전 단계의 데이터로 차단과 수거 자원의 배치를 안내하여 연안 폐기물 수거와 처리의 효율을 높인다. 야간과 악천후에 관측이 어려운 드론과 위성의 한계를 상시 고정형 데이터로 보완한다. 민간 기술 비영리 단체와 지방정부가 협력하여 개인정보 위험을 낮추면서 환경 데이터를 공공자산으로 전환한 실증 사례를 남긴다.

## 3. 예상 우려와 대응 (공무원 검토 대비, 가장 중요)

공무원이 가장 먼저 검토하는 것은 지자체 예산이 드는지와 민원이나 개인정보 문제가 생기는지와 책임이 어디에 있는지다. 이 우려를 사업 초기부터 막도록 설계했다.

| 부산시의 우려 | 이타서울의 대응 |
|---|---|
| 지자체 예산이 드는가 | 설치와 운영비는 민간이 전액 부담한다. 일차 자본은 마이크로소프트 환경 기금에 신청 중이며, 부산시에는 점용 허가와 행정예고 같은 행정 협조만 요청한다. |
| 개인정보 민원과 유출 책임 | 얼굴과 번호판을 현장 엣지 단에서 실시간 마스킹하고 영상 원본을 서버에 저장하지 않아 위험을 크게 낮춘다. 운영 주체가 개인정보 보호법상 운영자 의무를 지며 운영 관리 방침과 안내판을 이행한다. 실증 뒤 기부채납을 택하면 운영자 책임이 지자체로 넘어간다. |
| 관제 인력이 부족하다 | 실시간 관제를 요청하지 않는다. 정제된 환경 데이터와 통계만 제공하므로 추가 관제 인력이 들지 않는다. |
| 하천 시설의 홍수와 안전 책임 | 시설은 소형이고 가역이며 비굴착으로 설치한다. 홍수기 철거와 원상 복구 절차를 상시 운영하고 책임 보험을 든다. 점용 허가의 치수와 통수 요건을 충족하도록 설계한다. |
| 민간이 중도에 철수할 위험 | 실증 기간은 협약으로 운영하고 지속 단계는 기부채납으로 설계한다. 도면과 데이터와 운영 절차를 공개하므로 지자체가 이어받을 수 있다. |
| 인공지능 데이터를 믿을 수 있는가 | 자동 정확 집계를 약속하지 않는다. 존재와 추세를 보여주는 실증으로 두고 현장 확인으로 신뢰도를 보강한다. |
| 어느 부서의 업무인가 | 해양정책과는 데이터 수요자이자 정책 활용자이고, 점용 허가는 관할 구청이 맡으며, 부서 사이 조율 부담은 이타서울이 진다. |

요지는 부산시가 부담하는 것은 행정 협조뿐이고 비용과 운영과 기술 책임은 이타서울이 지며, 개인정보 위험은 설계 단계에서 낮춘다는 점이다.

## 4. 검토 메모 (외부 자료에서 교정한 것)

공무원에게 보내는 문서라 과장과 미검증 표현을 다음과 같이 바로잡았다.

첫째 육상 기인 비율을 60퍼센트로 단정하지 않고 상당 부분으로 두고 수치는 부산시와 해양수산부 자료로 확정하도록 표시했다. 담당 부서가 실제 수치를 알고 있을 가능성이 높으므로 틀린 수치를 적는 위험을 피한다.

둘째 100퍼센트 비식별과 리스크 제로라는 표현을 쓰지 않았다. 마스킹과 원본 미저장으로 위험을 크게 낮춘다고 적되, 운영 주체가 개인정보 보호법상 운영자 의무를 진다는 점을 명시했다.

셋째 정밀 계측이라는 표현 대신 실증이며 자동 정확 집계를 약속하지 않는다고 적었다.

넷째 스마트오션빌리지라는 플랫폼 이름을 단정하지 않고 부산시 해양환경 데이터 플랫폼으로 두며 명칭과 연계 사양을 확인한 뒤 협의한다고 적었다.

다섯째 사업 대상 개소수와 대표자 성명은 발송 전 확정하도록 비워 두었고 규모는 약 열 대 내외로 보수적으로 적었다.
