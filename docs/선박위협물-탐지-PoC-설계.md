# 선박 위협물 탐지 PoC 설계 (AIHub 71293)

작성 2026.06.30. 목적은 둘이다. 첫째 해운·항만 사회가치경영 피치를 위한 **선박 안전 데이터 역량 데모** — 선박 추진기에 감기는 폐어구·로프·금속 같은 위협물을 영상에서 탐지·분류한다([[maritime-csv-funding]]). 둘째 SEA:CUT perception의 **0단계 부트스트랩** — 탐지 결과를 우리 관측 레코드(class_dist)로 적재해 학습 루프에 연결한다([[physical-ai-0stage-impl]]). 문체는 서술형, 과장 금지, PoC임을 분명히 한다.

## 0. 무엇을 증명하나

"우리는 청소만 하는 단체가 아니라, 선박을 위협하는 부유·수중 폐기물을 자동으로 탐지·분류하는 인공지능과 데이터 역량을 가진다." 이 한 줄을 KOBC·BPA에 숫자로 보여주는 것이 PoC의 목적이다. 학습·배포 모델이 아니라 역량 데모다.

## 1. 데이터 — AIHub 「하천 및 항만 수중생활 폐기물」(dataSetSn=71293)

- 10종 라벨: 낚시도구, 봉돌, 유리병, 음료캔, 통조림, PET병, 장갑, 의류, 고철, 마스크.
- **선박 위협물 부분집합(vessel-threat)**: 낚시도구(폐어구·로프류)·고철·의류(천·로프 유사)를 1순위 위협으로 플래그한다. 감김 사고의 핵심이 폐어구이기 때문이다.
- 형식: 이미지/영상 + 라벨(AIHub는 보통 커스텀 JSON 어노테이션). YOLO 형식(class x_center y_center w h, 정규화)으로 변환 필요.
- **정직한 도메인 갭**: 이 데이터는 수중·항만이고 우리 현장은 하천 수면 부유물이다. 그대로는 우리 붐에 전이가 약하다. 그래서 ①해운 피치용 위협물 탐지 데모로는 직접 유효(항만 수중 위협물 그 자체) ②우리 붐 perception으로는 사전학습 가중치로만 쓰고 합성·현장 데이터로 미세조정([[opensource-collection-robot]]).
- **라이선스**: AIHub 데이터는 이용 승인·약관 준수 대상. 공개 저장소에는 원본을 올리지 않고 변환 스크립트와 가중치만 둔다.

## 2. 모델 — YOLOv11n (엣지 검증됨)

- YOLOv11 nano. 검증: Jetson Orin Nano에서 640×640 약 120fps, 수면 쓰레기용 경량 YOLO가 Jetson Nano급에서도 실시간. 임베디드 친화.
- 학습: ultralytics 프레임워크, 입력 640, 사전학습 가중치(yolo11n.pt)에서 전이학습, 10 클래스.
- 클래스 불균형(마스크·장갑 희소 가능) 대응으로 증강과 클래스 가중을 둔다.

## 3. 평가

- 지표: mAP@0.5, mAP@0.5:0.95, **선박 위협물 부분집합의 클래스별 재현율 우선**(놓치면 사고이므로 재현율 중시).
- 혼동행렬로 위협물 대 비위협물 분리 확인.
- 데모 산출: 테스트 영상에서 "선박 위협물 N건 탐지(폐어구 a·고철 b…), 평균 신뢰도 c" 요약.

## 4. 엣지 내보내기 (배포 경로)

- ONNX 내보내기 → **Hailo(.hef)** 변환(Pi 5 + AI Kit, 1기 고정 스테이션) 또는 **TensorRT INT8**(Jetson Orin Nano, 2기 보트).
- INT8 양자화로 10MB 이하·실시간 목표(스터디 5장 칩 정합).

## 5. 우리 시스템 연결 (PoC를 제품으로 잇는 다리)

- 탐지 결과 → 우리 관측 레코드의 `class_dist`(plastic/styrofoam/wood/other 매핑)와 위협 플래그로 적재(`/api/observations`). 71293 10종을 우리 4종 + vessel_threat 불리언으로 매핑한다.
- 누적되면 수거 중량 앵커(`/api/observations/anchor`)로 보정 → 학습형으로([[physical-ai-0stage-impl]]).
- 즉 이 PoC는 해운 데모이자 우리 perception의 첫 모델 버전(`model_ver=det-v0`)이다.

## 6. 산출물 — KOBC·BPA 피치 자료

- 한 장: 입력 프레임 → 위협물 박스·라벨 오버레이 → "선박 위협물 자동 탐지" 캡션 + 지표.
- 메시지: 우리 데이터·AI 역량이 귀 기관의 선박 안전·해양쓰레기 사회가치경영에 직접 닿는다.

## 7. 정직한 한계

- 수중·항만 데이터라 하천 수면 도메인 갭(5장 미세조정 필요).
- PoC는 역량 데모이며 배포·정확 집계를 약속하지 않는다.
- AIHub 라이선스 준수, 원본 비공개.
- 위협물 부분집합은 휴리스틱 매핑이며 현장 검증으로 보정.

## 8. 스캐폴드 위치

`poc/vessel-threat-detection/` 에 미검증 스캐폴드를 둔다. 데이터 연결 후 실행한다.
- `classes.yaml` — 10 클래스 + 선박 위협물 플래그 + 우리 4종 매핑.
- `prepare_aihub.py` — AIHub JSON 어노테이션 → YOLO 형식 변환 스텁(실 스키마는 받은 샘플로 확정).
- `train.py` — ultralytics YOLOv11n 전이학습 + 평가 + ONNX 내보내기.
- `README.md` — 실행 절차·라이선스·도메인 갭 경고.

## 9. 다음 스텝

1. 받은 71293 샘플의 실제 라벨 스키마 확인 → `prepare_aihub.py` 확정.
2. YOLOv11n 전이학습 1회 → 위협물 클래스 재현율 확인.
3. 위협물 탐지 오버레이 1장으로 KOBC·BPA 피치 자료 작성.
4. 탐지→관측 레코드 매핑 어댑터로 0단계 루프 연결.
