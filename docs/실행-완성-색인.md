## 2. 발송 패키지 — 최종 상태

세 갈래 모두 본문 최종본 완료. 발송은 빈칸만 채우면 즉시 가능.

- **부산시** — 풀버전 메일(`docs/부산시-해양정책과-제안-메일초안.md` 5절) + 사전미팅 요약 메일(8절) + 격식 공문(7절). 수신=해양수도정책과 해양환경팀.
- **KOBC·BPA** — 정렬 제안 메일·BPA 변형·수신 창구(`docs/해운-항만-CSV-제안-초안.md`). 근거=`docs/해운업계-CSV-후원-근거.md`.
- **빈칸(의뢰인만 채울 수 있음)**: 받는 팀장/담당자 성함, 5월 31일 정화 장소, 보내는 담당자 서명·연락처. (그 외 전부 확정.)

## 3. 엔드포인트 색인 (라이브)

| 엔드포인트 | 기능 |
|---|---|
| `GET /api/observations?site=` | 실데이터(부산 강우·HRFCO)로 관측 레코드 조립(`format=jsonl`) |
| `POST /api/observations` | 레코드 검증·JSONL 적재(`OBS_SINK_PATH`) |
| `POST /api/observations/anchor` | 수거 중량을 예측 레코드에 결합(학습쌍), `GET`=누적 학습셋 |
| `GET /api/litter-risk` | 지금 이 시간 퇴적 위험 핫스팟(`?log=1` 적재, `?tideFactor=` 조위, `?format=observations`) |
| `GET /api/khoa-tide` | 국립해양조사원 실시간 조위(기본 가덕도 DT_0063, 키 필요) |
| 기존 | `/api/hrfco` `/api/busan-rain` `/api/cctv` `/api/kma` `/api/seoul` `/api/wrn` |

지도(`NakdongMap`)는 위 litter-risk를 후광 링·팝업·라이브 요약으로 표시.

## 5. 잔여 의뢰인 액션 (최소)

1. **KHOA 조위 — 새 키 불필요.** 기존 `KMA_SERVICE_KEY`로 data.go.kr #15142507(조위관측소 실측·예측) 활용신청 후, 요청 URL을 `KHOA_DATAGO_URL`(토큰 {key}{obs}{date})에 설정 → 가덕도 조위 자동 라이브. (대안: 바다누리 무료 키 `KHOA_KEY`.)
2. **메일 3건 발송** — 빈칸(성함·5/31 장소·서명) 채워 부산시 해양수도정책과·KOBC·BPA로.
3. **수거 중량 입력 시작** — 정기 수거 때 `POST /api/observations/anchor`로 첫 라벨 → 학습 루프 가동.
4. **괴정천 단면 측정** → 컨베이어 폭·인양 높이 확정 → 레벨 C 조달(감전천 A 먼저).

근거·아키텍처 총괄은 `docs/SEA-CUT-전략-로드맵-1페이지.md`.
