# GPS 드리프터 표류 실측 — litter-risk 정답 라벨을 저비용으로 만드는 법

2026-06-30, 의뢰인이 "GPS ball 방식이 더 나을 수도 있고, The Ocean Cleanup이 이미 했을 수 있으니 분석하라"고 제기. 원문 5건을 직접 확인해, GPS 드리프터(추적 부표/병)가 SEA:CUT의 표류 예측에서 무엇을 하는지, 우리 코드·자산에 어떻게 붙는지 매핑한다. 핵심 결론부터: **GPS 드리프터는 탐지·예측을 대체하는 게 아니라, 우리 `/api/litter-risk` 표류 예측의 "정답 라벨(ground truth)"을 저비용·오픈소스로 생산하는 도구다.** 자매 문서 [[글로벌-AI-정화사례-벤치마킹-심층스터디]]가 "탐지→표류→경로" 풀스택을 다뤘다면, 이 문서는 그 표류 단계의 **검증 데이터를 어떻게 만드나**를 다룬다. 수치는 원문 검증분만, 과장 금지.

---

## 0. 한 줄 합의 — 왜 "GPS ball이 더 나을 수도" 있나

자매 문서의 ADOPT·DEEP-PLAST는 위성·AI로 표류를 *예측*했지만, 두 연구 모두 **"표류 검증이 ground truth 부족으로 정성적"**이라고 정직하게 한계를 밝혔다. 바로 그 공백을 메우는 게 GPS 드리프터다 — **쓰레기가 실제로 어디로 흐르는지를 좌표 시계열로 실측**한다.

SEA:CUT에 특히 맞는 이유는 위치와 규모다:

- **고정 붐 = 드리프터의 종점.** 외해는 드리프터를 잃지만, 소하천은 짧고 폭이 좁아 **드리프터가 결국 우리 붐에 걸린다.** 발생원(투하점)→붐(회수점) 전 경로가 한 번에 라벨이 된다.
- **회수·재사용 가능.** 외해 dummy plastic은 회수 불가(그래서 윤리 논란)지만, 소하천 드리프터는 회수해 재사용 → 비용·윤리 동시 해결.
- **이미 한국 하구에서 방법론이 섰다.** 제주 4개 하구 연구(동아대, 2025)가 발생원 역추적 공식(VSC·풍속계수)을 검증했고, The Ocean Cleanup이 2016년부터 선례를 쌓았다.

즉 "GPS ball이 더 낫다"가 아니라, **GPS ball이 우리 예측기의 정답지를 만들어 예측을 학습형으로 끌어올리는 결정적 보완재**다. 우리 학습 루프([[physical-ai-0stage-impl]])로 보면: 수거 중량 앵커가 "얼마나"의 라벨이라면, **드리프터 궤적은 "어디로·어떻게"의 라벨**이다.

## 2. 사례별 심층

### 2.1 Message in a bottle — 오픈하드웨어·오픈데이터의 교과서

**무엇인가.** Exeter대 Emily Duncan 외, *PLOS ONE*(2020-12-02, DOI 10.1371/journal.pone.0242459). 내셔널지오그래픽 "Sea to Source: Ganges" 탐사의 일부.

- **하드웨어:** Arribada Horizon GPS 보드(세라믹 안테나) + ARGOS ARTIC R2 위성 송수신기 또는 GSM 셀룰러 모뎀. **현지에서 수집한 500ml PET 음료병을 재사용**해 전자장치를 캡슐에 넣되 원래 부력·형태 유지 → 실제 플라스틱의 표류 패턴 모방.
- **전력:** GSM 모드(9mA, 약 8주) vs 위성 모드(**38μA, 9년+**) — 약 1,100배 차이.
- **배치·결과:** 총 25개(인도 하천 10 + 방글라데시 하천 12 + 해상 3). 하천 평균 추적 20~23일, **최대 609.9km**. 해상은 동인도 해안 해류 따라 서향, **최대 2,845km·94일**. 하천 병의 **40%가 제방에 고착**(stepwise displacement).
- **오픈 공개:** 데이터는 **Movebank 오픈 저장소**(study ID 1271155477), 하드웨어·소프트웨어는 **GPLv3 & CERN 1.2 오픈소스 라이선스**(Icoteq/Arribada).

**가져올 것.** ① **PET병 재사용** = 우리 "실제 부유물과 같은 부력으로 표류 모방" 원칙. ② 오픈하드웨어/오픈데이터 = 우리 공개 전략([[public-agency-focus]] 데이터 개방 통로)과 정합 — Movebank·Zenodo 같은 표준 저장소에 궤적 공개. ③ **하천 고착률 40%**는 우리에게 결정적 통찰 — 소하천 쓰레기는 상당수가 제방·식생에 걸린다 → 우리 litter-risk의 "트랩(trap)" 항을 실측으로 보정할 수 있다.

**거를 것.** 위성(ARGOS) 모드는 외해용. 소하천은 셀룰러(NB-IoT/LTE-M)로 충분하고 더 싸다.

### 2.2 제주 4개 하구 드리프터 — 한국 정합 + 발생원 역추적 공식

**무엇인가.** 동아대(에너지·광물자원공학과) Kang 외, *대한원격탐사학회지* 41(5):911-925(2025-10, DOI 10.7780/kjrs.2025.41.5.16). **한국 하구에서 부유물 발생원을 GPS 드리프터로 역추적**한 직결 사례.

- **하드웨어:** GNSS(정확도 2.5m, CER50%) + 다중망(NB-IoT/LTE-M/GPRS) + 오픈소스 MCU + 충전식 리튬이온. **1시간 위치 수집, 6시간 전송**, 클라우드 자동 저장. 위성(Iridium) 대비 저비용·저전력.
- **배치:** 방위별 4개 하구 — 천미천(동)·금성천(서)·효돈천(남)·화북천(북), 2025년 7~8월 하계 강우기, 총 9개 buoy.
- **방법론(핵심).** 궤적을 풍향·해류와 동시 분석:
  - **벡터유사계수(VSC):** 궤적 vs 해류·풍속 방향 일치도.
  - **풍속계수 α** = |실제드리프트속도 − 해류속도| / 풍속.
  - 데이터: NOAA NOMADS(풍, 0.25°/6h) + CMEMS(해류, 0.083°/6h) + 기상청(강우, 4지점).
- **결과:** 천미천은 외해로 **137.78km 장거리 확산**, 효돈천은 하구 **34~62m 내 정체**(높은 지역 축적). 해류가 풍속보다 우위(α=0.012~0.027로 풍 영향 미미).

**가져올 것 — 가장 직결.** 이건 **우리가 그대로 이식할 수 있는 한국형 방법론**이다. ① **발생원 역추적 = 우리 litter-risk의 역방향 검증**(예측은 발생원→퇴적, 드리프터는 퇴적→발생원 확인). ② VSC·α 공식으로 "강우 vs 풍 vs 유속 중 무엇이 표류를 지배하나"를 정량 분리 → 우리 휴리스틱(발생원×트랩×강우펄스)의 가중치를 실측 보정. ③ 동일 통신(NB-IoT/LTE-M)·1h/6h 주기를 우리 IoT 설계 기본값으로 채택. ④ 제주 2021년 해양쓰레기 22,082톤이라는 규모 근거는 부산 사례 제안에도 인용 가능.

### 2.3 The Ocean Cleanup — 'hotlines' 개념과 글로벌 선례

**무엇인가.** TOC는 2016년 네덜란드·미얀마에서 GPS 추적기를 처음 테스트(자체제작 GPS 수신기 + 스마트폰 두 종). 추적기들이 "쓰레기 풍부한 해안선에 도달"하며 **'hotlines'(강에서 플라스틱이 집중 축적되는 경로)**를 식별. 이후 2018년 20개+ 강 측정, 미시시피 Baton Rouge에서 **GPS 내장 병 파일럿**("30 cities" 프로그램), 자카르타·과테말라 모타과강으로 확장. 이 데이터가 **글로벌 강 플라스틱 모델과 Interceptor 배치**의 기초가 됐다.

**가져올 것.** ① **'hotlines' 개념 = 우리 핫스팟의 실측 근거.** 우리는 litter-risk로 핫스팟을 *예측*하지만, TOC는 드리프터로 핫라인을 *실측*해 모델을 세웠다 — 같은 길을 소하천 규모로. ② 드리프터 데이터 → 차단시설(Interceptor) 배치 결정. 우리도 **드리프터 궤적으로 OpenBoom 최적 설치 지점을 정당화**할 수 있다(어디에 쳐야 가장 많이 걸리나). ③ "이미 글로벌 1위 조직이 한 검증된 방법"이라는 점이 제안·규제 설득의 무게.

**거를 것.** dummy plastic을 일부러 외해에 흘리는 방식은 회수 불가라 윤리 비판이 있다 — 우리는 **회수 전제 + 재사용병**으로 그 함정을 피한다.

### 2.4 저비용 Arduino 드리프터 + Arno Smart Drifter — 제작 경로와 운영 단순성

- **Arduino 드리프터**(*Geoscientific Instrumentation* 9:435, 2020): off-the-shelf Arduino + GPS 수신기 + 셀룰러 모뎀 + 로컬 저장으로 **전기공학 배경 없이 제작 가능**, 전체 코드 GitHub·Zenodo 공개. 일반 Arduino GPS 트래커 BOM은 ~$25–35 수준(추정, 일반 부품가 기준).
- **Arno강 Smart Drifter Cluster**: 소비자 IoT로 실제 해양 쓰레기 궤적 추적, "신뢰·견고·**자가전원·거의 무유지보수**"를 핵심 설계 가치로.

**가져올 것.** 우리 1기 저비용 철학([[opensource-collection-robot]] Pi5+Hailo $70)과 정확히 같은 결 — **드리프터도 오픈하드웨어로 자작**해 BOM을 수만 원대로 낮춘다. 무유지보수·자가전원은 시민 운영 전제에 필수.

## 3. 왜 SEA:CUT에 GPS 드리프터가 특히 맞나 — 고정 붐과의 결합

외해 사례 대비 **소하천 + 고정 붐**이라는 우리 조건이 드리프터에 이상적이다:

1. **종점이 정해져 있다.** 외해 드리프터는 어디로 갈지 몰라 잃는다. 우리는 **드리프터가 결국 하류 붐에 걸린다** → 100% 가까운 회수율, 발생원→붐 전 경로 라벨.
2. **거리가 짧아 통신·배터리 부담이 작다.** 갠지스 609km가 아니라 수 km 구간 → 저가 셀룰러로 충분, 며칠~몇 주면 종료.
3. **붐 자체가 reference frame.** 같은 붐에 여러 드리프터가 어디서·어떻게 도착하는지 누적하면, **붐 단면의 유입 패턴 지도**가 생긴다 → litter-risk 예측의 직접 정답.
4. **예측-실측 폐루프.** litter-risk가 "이 강우에 A지점 쓰레기가 B퇴적지로 갈 것"이라 예측 → 드리프터로 실제 경로 확인 → 오차로 모델 보정. 이게 [[physical-ai-boom]]의 "인과·폐루프 데이터" 그 자체.

## 6. 한국 소하천·하구 정합 + 규제·윤리

- **국내 선례 존재.** 제주 동아대 연구가 **한국 하구에서 이미 방법론을 검증** → "해외 것 들여온다"가 아니라 "국내 검증된 방법의 소하천 적용"으로 명분이 선다.
- **회수 전제 = 윤리 통과.** TOC의 외해 dummy plastic 논란을 피한다 — 우리 드리프터는 붐에서 회수·재사용하므로 "쓰레기를 일부러 버린다"는 비판 무효.
- **규제.** 하천 점용·실증은 기존 트랙([[cctv-data-collab-model]], [[permit-agent]]) 안에서 처리. 드리프터는 무동력 부유물이라 추가 규제 부담이 작다(단 어구·항행 안전 구간은 사전 협의).
- **비식별 무관.** 드리프터는 사물 추적이라 영상 비식별([[public-agency-focus]]) 이슈에서 자유 — 오히려 깨끗한 데이터.

---

## 7. 정직 경계 — "대체"가 아니라 "보완"

- **드리프터는 탐지·예측을 대체하지 않는다.** 붐 카메라(상시 계측)·litter-risk(예측)는 그대로 가고, 드리프터는 **간헐적 캘리브레이션 캠페인**으로 정답을 공급한다. 상시 운영 도구가 아니다.
- **표본 한계.** 드리프터 몇 개의 궤적은 전체 쓰레기 흐름의 표본일 뿐. 제주도 9개, 갠지스 25개였다 — 통계적 일반화엔 반복 투하 필요.
- **소하천 특수 리스크.** 좁고 얕은 소하천은 갈대·바위에 조기 고착하거나 통신 음영(계곡)에 빠질 수 있다 → 회수는 쉽지만 데이터는 끊길 수 있음.
- **타 도메인 수치 직접 인용 금지.** 갠지스 609km·고착 40%, 제주 137km는 **그 하천의 값**이다. 우리 소하천 결과로 인용하지 않는다([[글로벌-AI-정화사례-벤치마킹-심층스터디]]·[[maritime-csv-funding]]의 직접귀속 금지 규율과 동일).

---

## 8. 리스크와 미해결

- **회수 실패 시 우리도 쓰레기 배출자가 된다** → 회수율·재사용을 운영 KPI로 명문화, 분실 대비 생분해 하우징 검토.
- **통신비·운영 인력.** 시민 캠페인으로 묶지 않으면 반복 투하·회수가 부담 → [[opensource-collection-robot]] 시민 트랙과 결합.
- **litter-risk와의 정합 검증 미실시.** 현재는 설계 제안 — 실제 예측-실측 오차는 시제품 투하 후에야 나온다.

## 9. 즉시 행동 (다음 스텝)

1. `src/lib/observation.ts`에 `drift_track` 레코드 타입(또는 `track_id` + 좌표 시계열) 설계 초안 — 앵커(`collected_mass_kg`)의 공간 짝으로.
2. 제주 동아대 방법론(VSC·풍속계수 α)을 `/api/litter-risk` **검증 지표**로 PLAN.md에 반영(예측 핫스팟 vs 실측 궤적 비교).
3. 드리프터 BOM(셀룰러·NB-IoT 기반, 재사용 PET병)을 `/hardware` 저장소 구조에 신규 항목으로 — 1대 수만 원대 목표, 단가 시제품으로 확정.
4. ChangeX·규제샌드박스 소구문에 "국내 검증(제주 동아대)된 드리프터 방법을 소하천 발생원 추적에 적용 + 회수·재사용으로 윤리 통과" 한 줄 추가.
