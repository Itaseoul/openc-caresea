# 유튜브 톤 가이드 + 슈퍼챗 설계 — "예측→안전수거→투명데이터"

작성 2026.06.30. 이타 자문 이사회 검토를 반영한 (b) 산출물이다. 두 가지 위험 요소를 도려냈다. "재난 구조 생중계" 톤(디재스터테인먼트 시비·평판 리스크)과 "부스터 슈퍼챗으로 로봇 모터 출력을 올려 조종"(유료 실시간 원격조종 = 배상책임 폭탄, 지난 검토에서 막은 제안 D의 재발). 남는 자산은 충분히 강하다. 긴장감은 *재난쇼*가 아니라 *"비 오기 전에 어디 쌓일지 맞히는 예측 서사"*에서 뽑는다.

## 0. 채널 콘셉트 재정의

기존 제안의 "전국 하천 쓰레기 저지선 / 재난 구조 생방송"을 **"예측하고, 안전하게 치우고, 데이터를 투명하게 보여주는 다큐멘터리 채널"**로 바꾼다. 시청자가 보는 클라이맥스는 "급류 속 로봇 출격"이 아니라 "예측이 맞아떨어지는 순간"과 "안전창에 시민·붐이 함께 치워낸 전후(before/after)"다.

## 1. 톤 가이드 — 금지어와 대체

금지: "재난 구조 생중계", "출격", "레이드", "급류 속으로", 실시간 위기 연출. 한국은 매년 호우로 실제 인명피해가 난다. 환경정화를 아드레날린 재난쇼로 포장하면 평판 리스크이고, 실제 위험 행동을 미화한다.

대체: "예측 라이브", "안전창 수거", "데이터로 보는 하천", "비 오기 전 점검". 긴장은 예측의 적중/오차에서 만든다 — "지난주 우리가 위험하다고 표시한 그 만곡부, 오늘 가보니 정말 쌓였습니다" 같은 검증 서사.

원칙: 화면 어디에도 시청자가 실제 로봇을 실시간으로 조종하는 인터랙션은 없다. 시청자 참여는 **관전·예측 투표·비실시간 라벨링**까지다.

## 2. 콘텐츠 포맷 4종

예측 라이브(비 오기 전): `litter-risk` 예측 지도를 띄우고 "이번 강우에 어디가 위험한가"를 해설한다. 시청자는 핫스팟을 투표·라벨링한다.

안전창 수거 라이브(비 후, 수위 안전선 아래): 정지형 붐이 모아둔 부유물을 시민·동호회가 안전하게 수거하는 전후를 중계한다. 로봇 카메라의 YOLO 바운딩 박스 화면은 *관전용*으로 한쪽에 띄운다(조종 아님).

팩토리 라이브: 3D 프린터가 부품을 뽑고 에지 칩이 흙탕물 속 플라스틱을 분별하는 제작 과정. 저비용·고서사.

드리프터 추적 라이브: GPS 드리프터를 띄워 "쓰레기가 실제로 어디로 흐르는가"를 실시간 궤적으로 보여준다(`GPS-드리프터-표류실측-심층스터디.md`). 예측의 정답을 눈앞에서 검증하는 가장 강한 그림.

## 3. 슈퍼챗 설계 — 모금은 OK, 조종은 금지

허용. 슈퍼챗은 **모금·응원·매칭 트리거**로만 쓴다. 화면엔 "이 슈퍼챗이 필라멘트 N롤 / 예비 배터리 N개 / 센서 N점"으로 환산해 표시한다(현물 환산 효능감, 조종 효과 아님). "기업 매칭 릴레이"(시리얼스 클럽 멤버나 후원 기업이 라이브에 들어와 시청자 슈퍼챗 금액만큼 매칭 선언)는 그대로 살린다 — 참여 폭발의 핵심이고 안전하다.

금지. "만원 슈퍼챗 = 모터 120% / 저 쓰레기로 이동"식 **유료 실시간 조종**은 전면 불허. 공공수역 로봇을 돈 받고 시청자가 조준·이동시키면 사고 시 *유료 지시*라 배상책임이 가중된다. "배터리 충전 펀딩" 게이지는 모금 연출로는 가능하나, 그 결과로 *현장 로봇이 실시간으로 움직이는* 연결은 끊는다.

대안 인터랙션. "이 슈퍼챗으로 다음 안전창 수거 1구역 추가" 같은 **비실시간 약속**은 가능하다(현장 즉시 구동이 아니라 다음 일정에 반영). 시청자의 핫스팟 라벨링이 다음 회차 예측·수거 계획에 반영되는 것을 보여주면 효능감은 유지되고 위험은 0이다.

## 4. 법적·정직 경계

슈퍼챗 수익을 "기부"로 다루는 순간 기부금품법·세무(지정기부금)·플랫폼 정책의 경계가 생긴다 — 슈퍼챗은 유튜브 수익 배분이지 직접 기부가 아니므로, "모금"과 "후원 안내"의 표현을 분리해야 한다. 발송 전 ⑫ 공익법인 자문 1건 선결(`헤일로요트-콘셉트vs조선사-분기브리프.md`의 자문 항목과 묶어 처리).

임팩트 숫자는 실측만 단정한다. 수거량은 kg 실측, 탄소·구조된 생물 수 등은 *추정·범위*로 표기하고 직접 귀속하지 않는다(메모리 정직 원칙).
